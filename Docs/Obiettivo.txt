L'obiettivo del progetto è quello di creare un agente in grado di organizzare vari tipi di documenti, per il momento testuali, in varie cartelle senza che l'utente debba fare alcun intervento, eccezion fatta per la fornitura dei vari documenti. 

Il sistema dovrà dunque cercare di costruire le relazioni tra i vari documenti e cercare il modo migliore di organizzarli, a seconda del contenuto. Questo implica quindi la creazione automatica di cartelle, inserendo all'intenrno i documenti correlati. 

I tipi di file da categorizzare comprenderanno le seguenti estensioni:

	.Pdf 
	.txt
	.tex, con i suoi vari file aggiuntivi
	.tar
	.file di codice (la maggior parte in python)
	.html (pagine web)
	.xlsx
	.docx

Quindi essenzialmente avremo dei documenti scientifici, degli appunti universitari, dei programmi, dei progetti personali e delle pagine web. Allo stato dell'arte, l'agente dovrebbe creare delle cartelle separte per ogni argomento, classificando i vari doucmenti in base alla loro cateoria dia ppaprtenzna e separandoli in varie sottocartelle. Ciò implica hce deve costruire anche delle relazioni tra file, considerando che questi potrebbero avere estensioni differenti. 

I criteri di organizzazione saranno dunque i seguenti: 

	1. Si parte dal file raw, quindi un insieme dismogeneo di file e documenti, dopodiche vi è una prima classificazione tramite le estensioni, successivmaente si cominciano a metter ein relazione i vari file tramite l'insiemrenti id questi in cartelle e sottocartelle (opzionali).
	2. La classificaizone avverrà seguendo dei pattern in comune, nel senso che il modello richerrchera le possibili relazioni in base agli elementi in comune tra i vari documenti. Per esempio: i file .tex saranno affiancati sempre da un .aux o .toc, e qiundi andranno messi insieme, all'interno del .tex ci sarà sicuramente la data che potrebbero un primo indice di classficazione; ci sarà sicuramente anche un file .pdf generato, che dovrà o essere collegato inqualhce modo alla cartella oppure inserito direttamente inquest'ultima. 
	3. Nel caos in cui il nome del documento dovesse essere fuorviante o addiritutra completamente distaccato dal suo contenuto, e quindi inutile per la classificazione, si procederà con la misura della similarità con gli altri documenti
